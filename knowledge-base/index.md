# Knowledge Base Index

A guide to what is in the knowledge base and where to find it. Agents should check here before searching individual folders.

---

## Brand

**`brand/voice-rules.md`** - Active. Global writing rules for every agent and every output. Read this before producing any written content.

**`brand/visual-rules.md`** - Placeholder. Brand colors, fonts, and visual guidelines. Not yet filled in. Until real brand assets are provided: default to clean layouts, do not invent colors or fonts, and flag any creative brief that requires specific brand assets.

---

## Frameworks

**`frameworks/funnel-math.md`** - Active. Reverse-engineered funnel math model. Work backward from revenue goal to leads needed to required ad budget. Includes benchmark conversion rates, minimum viable budget rule, and ROAS sanity check.

**`frameworks/creative-testing.md`** - Active. Five-level creative testing hierarchy (Format → Concept → Hook → Color → Copy). Kill and scale thresholds, one-variable-at-a-time rule, naming convention.

**`frameworks/offer-formula.md`** - Active. Offer building framework. One-sentence offer test, four offer elements, offer value equation, value stack rules, guarantee construction types, urgency and scarcity rules.

**`frameworks/ad-diagnosis.md`** - Active. Strict metrics read order for diagnosing ad performance: Impressions → Frequency → CTR → CPC → LP CVR → CPL → Lead quality. Common pattern diagnoses and 5-decision framework (SCALE/KEEP/ITERATE/KILL/TEST).

**`frameworks/content-pillars.md`** - Active. Five content pillar system with percentage splits (Education 40%, Social Proof 20%, Brand/Story 20%, Engagement 10%, Offer 10%). Weekly rhythm template and platform-specific format notes.

---

## SOPs

**`sop/client-onboarding.md`** - Active. 14-day new client onboarding process from kickoff call to campaign live. Covers all owners, milestones, and handoffs. First 30-day check-in schedule included.

**`sop/campaign-launch.md`** - Active. Campaign launch checklist and sequence. Pre-launch requirements across strategy, copy, creative, technical, and QA. Post-launch 48-hour monitoring and first optimization window rules.

**`sop/monthly-review.md`** - Active. End-of-month review process. Eight steps from data pull through next actions. Covers KPI review, funnel analysis, lead quality review, and ad performance review.

---

## Swipe Files

**`swipe-files/hooks.md`** - Active. Hook structure library by type: curiosity gap, pain mirror, proof/result, contrarian, bold claim, direct benefit. Includes platform-specific usage notes and email subject line variants.

**`swipe-files/subject-lines.md`** - Active. Email subject line structures by type with preview text directions. Rules for subject line length, formatting, and what causes subject lines to fail.

---

## Examples

**`examples/ad-copy-structures.md`** - Active. Five ad copy structures with format explanations and filled-in examples: PAS short form, proof-first story, contrarian hook, VSL spoken hook, direct response short form. Also includes headline structures and CTA examples.

## Clients

**`clients/client-template.md`** - Template. Copy this and fill in for each new client. Covers business overview, the offer, target audience, goals and KPIs, current situation, campaign context, assets, communication, and notes.

No client profiles added yet. Add a file using `client-template.md` when a new client is onboarded.

## Offers

No offer docs added yet. Add a file using `offer-template.md` for each active offer.

No case studies added yet. Add a file using `case-study-template.md` for each client result.

## Incoming Docs

**`incoming-docs/`** - Staging area. Drop raw files here before reviewing and filing.

Do not reference incoming docs from agent prompts. Review, extract, and move to the right folder before use.

---

## How To Update This Index

When you add a new file to any knowledge base folder:
1. Add a line to the relevant section above.
2. Note the file name, a one-line description, and whether it is Active, Template, or Placeholder.
3. Keep descriptions short enough to scan quickly - one line per file.
