# Webinar Script

## Purpose

Write a full live or automated webinar script. A webinar script covers the intro, teaching content, case studies, offer reveal, and close. The output is a complete word-for-word script with speaker notes, slide cue marks, and a timing guide. It is different from a VSL - a webinar has a longer teaching section and can run 60-90 minutes.

## Used By

- Copywriter (primary)
- Video Agent (if the webinar will be pre-recorded)
- Marketing Director (reviews before client sign-off)
- QA Agent (reviews for claims and brand voice)

## Trigger

Use when a client is launching a webinar as part of their funnel, or when a recorded webinar needs to be written or rewritten.

## Inputs Needed

- Offer name, price, and main promise - required
- Audience profile and awareness level - required
- Core teaching topic (what the webinar teaches) - required
- 2-3 case studies or client results - required
- Objections to overcome (from objection research) - required
- Webinar goal: sell from the stage, book calls, or grow email list - required
- Intended length (standard: 60-75 minutes) - helpful

## Process

**Step 1: Clarify the structure before writing**

A webinar script has five phases. Confirm the goal of each phase before writing content for it:

1. **Hook and intro** (5-10 min): Stop drop-off. The first 10 minutes have the highest abandonment. Promise a specific result and establish credibility quickly.
2. **Framework delivery / teaching** (25-35 min): Deliver on the promise. Teach enough to prove expertise and make the viewer trust the solution, but do not give everything away. Reveal the "what" and "why", not the full "how" - that is what the offer provides.
3. **Case study section** (10-15 min): Show real people getting results. Each case study should match a segment of the audience watching.
4. **Offer reveal** (10-15 min): Present the offer with a value stack, price anchor, and guarantee. Use the same structure as a VSL offer reveal.
5. **Q&A and close** (5-10 min): Handle objections live or via pre-loaded FAQ. Drive urgency to act now.

**Step 2: Write the hook and intro**

Three things must happen in the first 5 minutes:
1. Tell them what they are going to learn (specific outcome, not vague)
2. Make them believe they can get it (relatability, not just credibility)
3. Create a reason to stay (preview a story, stat, or case study result they will not want to miss)

Do not start with "Welcome, I'm so glad you're here." Start with the result or the problem.

Hook structure:
"By the end of this [webinar/training/workshop], you're going to know [specific outcome]. And I'm going to show you [mechanism name] that [specific type of person] is using to [result]."

**Step 3: Write the teaching section**

Format the teaching as a numbered framework with a name the audience can remember.

Each section of the framework:
- Makes one specific point
- Includes an example or mini case study
- Has a "so what" - why this matters to the viewer

Avoid: teaching so much that the viewer no longer needs the offer. Teach enough to create belief in the mechanism, not enough to replace buying it.

Slide cue format: [SLIDE: Title of slide] before each slide transition.

**Step 4: Write the case studies**

Each case study follows this format:
1. **Before**: Who was this person, what was their situation, what had they tried
2. **The shift**: What they did differently
3. **After**: Specific result with real numbers where possible
4. **Mirror**: One line connecting their story to someone watching ("If you're running a service business stuck under $15k/month, this is for you")

Use 2-3 case studies. Vary the profile so different audience segments see themselves.

**Step 5: Write the offer reveal**

Use the same value stack format as the VSL:
- What is included (name each component, do not list them as features)
- What each component is worth on its own
- Total value, then the actual price
- Price anchor (what comparable alternatives cost or what the problem costs them)
- Guarantee (specific, not generic)
- Fast-action bonus if using one (must be genuine, not a fake timer)

**Step 6: Write the close**

After the offer reveal:
1. Recap the decision: "You have two choices right now..."
2. Handle the top 3 objections directly ("The most common thing I hear at this point is...")
3. Final CTA with specific instruction (go to this URL, click this button, etc.)
4. Urgency if real ("This pricing is only available until [date/time]")

**Step 7: Add timing marks**

Label each section with approximate timing:
- [0:00-0:05] Hook and intro
- [0:05-0:35] Framework delivery
- [0:35-0:50] Case studies
- [0:50-1:05] Offer reveal
- [1:05-1:15] Close and Q&A

## Output Format

```
WEBINAR SCRIPT - [Client/Offer] - [Date]

WEBINAR TITLE: [Title]
GOAL: [Sell from stage / Book calls / List growth]
TARGET LENGTH: [X minutes]
AUDIENCE: [Brief description]

---

[0:00-0:05] HOOK AND INTRO

[SLIDE: Title slide]

[Script - word for word]

[SLIDE CUES embedded where needed]

---

[0:05-0:35] FRAMEWORK DELIVERY

FRAMEWORK NAME: [Name the framework]

[SLIDE: Framework overview]

SECTION 1: [Point name]

[Script]

[SLIDE: Section 1 slide]

[Continue for each framework point]

---

[0:35-0:50] CASE STUDIES

CASE STUDY 1: [Name or persona]

Before: [Script]
The shift: [Script]
After: [Script]
Mirror: [Script]

CASE STUDY 2: [Repeat]

---

[0:50-1:05] OFFER REVEAL

[SLIDE: Offer name]

[Script - full offer reveal with value stack, price, anchor, guarantee]

---

[1:05-1:15] CLOSE AND Q&A

[Script - decision recap, objection handling, final CTA, urgency]

---

SLIDE COUNT: [X total slides]
SCRIPT NOTES: [Any instructions for the presenter or video production team]
```

## Quality Check

- [ ] Opening 5 minutes do not start with "Welcome" or gratitude - they start with the result or problem
- [ ] Teaching section uses a named framework
- [ ] At least 2 case studies with real before/after structure
- [ ] Offer reveal includes value stack, price anchor, and specific guarantee
- [ ] Top 3 objections are handled in the close
- [ ] Slide cues are embedded throughout
- [ ] Timing marks are included
- [ ] No em dashes, no banned words, contractions used throughout
- [ ] QA Agent reviews claims section before sign-off
