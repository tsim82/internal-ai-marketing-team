# VSL Scene Plan

## Purpose

Produce the production-ready scene-by-scene breakdown for a Video Sales Letter (VSL). The VSL scene plan is separate from the VSL script (see copywriting/vsl-script.md). The script covers the spoken copy. The scene plan covers how each section of the VSL is produced: what is on screen, what B-roll plays, what text overlays appear, and how long each scene runs. The output is what a video editor uses to produce the final VSL.

## Used By

- Video Agent (primary)
- Copywriter (uses this to ensure script sections align with scene intent)
- Creative Design Agent (creates graphic overlays specified here)

## Trigger

Use when a VSL script is complete and the team is moving into production, or when an existing VSL needs to be restructured for better retention or conversion.

## Inputs Needed

- Completed VSL script (from copywriting/vsl-script.md) - required
- Offer and audience - required
- Production format (talking head, voiceover + slides, HeyGen avatar, animated) - required
- Available visual assets (product images, client photos, testimonial videos) - helpful
- Target VSL length - required

## Process

**Step 1: Identify the VSL sections from the script**

Standard VSL sections (map each to the script):
1. Hook (0:00-0:30)
2. Problem agitation (0:30-2:00)
3. Credibility bridge (2:00-3:00)
4. Mechanism introduction (3:00-5:00)
5. Case study / proof (5:00-8:00)
6. Offer reveal (8:00-11:00)
7. Value stack (11:00-13:00)
8. Price and anchor (13:00-14:30)
9. Guarantee (14:30-15:30)
10. FAQ / objection handling (15:30-18:00)
11. Close / CTA (18:00-20:00)

Timing varies. Adjust to match the actual script.

**Step 2: Assign a visual treatment to each scene**

For each scene, decide:
- **ON SCREEN**: What does the viewer see? (talking head, slides, graphic, B-roll, text only)
- **SPOKEN**: What section of the script plays here?
- **B-ROLL CUE**: If B-roll is used, describe the specific visual (specific, not generic)
- **TEXT OVERLAY**: Any text that appears on screen during this scene
- **PACING**: Fast / Medium / Slow - how is this scene edited?

B-roll cues must be specific. "Business owner working" is not specific. "Person typing on laptop at a standing desk in a modern office, overhead shot" is specific.

**Step 3: Plan the retention devices**

A VSL is typically 15-20 minutes. Without active retention devices, viewers drop off. Plan a retention device every 3-4 minutes:

- Switch from talking head to B-roll or graphic
- Cut to a testimonial clip
- Insert a bold text overlay with a key claim
- Switch camera angle or zoom
- Insert a short clip of the product or result

**Step 4: Plan the proof moments**

For the case study and testimonial sections:
- What specific proof assets exist? (video testimonials, screenshots, results images)
- Where does each piece of proof appear in the scene plan?
- If no video testimonial exists, what visual shows during the spoken testimonial?

**Step 5: Plan the offer reveal section**

The offer reveal is the most visually important section. Plan:
- Does a value stack graphic appear? (What does it show?)
- Does pricing appear as text on screen?
- When does the CTA button or link appear for the first time?
- Is there a countdown timer? When does it appear?

```
VSL SCENE PLAN - [Client/Offer] - [Date]

PRODUCTION FORMAT: [Talking head / Voiceover + slides / HeyGen / Animated]

TARGET LENGTH: [X minutes]

---

SCENE 1: HOOK
  Timing: [0:00-0:30]

  SPOKEN: [Script hook lines - first 3-5 sentences]
  B-ROLL CUE: [Specific description or N/A]
  TEXT OVERLAY: "[Key hook phrase on screen]"
  PACING: [Fast - hook must be snappy]
  RETENTION NOTE: [Hook must grab in first 3 seconds]
```

## Quality Check

- [ ] Every VSL section has a scene mapped to it
- [ ] B-roll cues are specific (not "show something relevant")
- [ ] Proof assets are inventoried (available vs needed)
- [ ] Retention device planned every 3-4 minutes
- [ ] Offer reveal scene specifies exactly when price and CTA appear
- [ ] Graphics needed are listed for Creative Design Agent
- [ ] Scene plan can be handed to a video editor without additional explanation
